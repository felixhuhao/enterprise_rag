# Smoke Test Checklist

用于每次检索链路、权限、前端布局调整后的轻量回归。除非特别说明，先用 Admin 用户跑一遍，再切到 Bob 或 Alice 验证普通用户视角。

## 1. 用户与导航

- Admin 左侧导航应显示：知识查询、文档管理、质量中心、检索测试、系统设置。
- 普通用户左侧导航应显示：知识查询、知识库文档、查询记录。
- 普通用户进入查询记录时，只能看到自己的查询记录和反馈范围。
- 普通用户不应看到检索测试、系统设置入口。
- 实体别名、用户管理、实体访问授权均已并入「系统设置」页的 Tab：实体别名、用户管理（Admin 可见）、实体配置（Admin 可见）。

## 2. 文档页面

- 文档管理列表可上传文档，状态筛选卡可点击过滤。
- 文档列表列宽默认可读：文件名占剩余宽度，状态/主体/切片数/图片/上传时间/操作不换行。
- 点击文档进入详情页，切片列表可搜索、分页、展开、复制。
- 文档详情应显示质量状态、质量摘要和可疑切片 warning 标签。
- 正常 Markdown 文档处理完成后，质量状态应为 `good` 或只有可解释的少量 warning。
- 上传并处理文档后，系统设置 -> 后台任务应出现对应 document ingestion job。
- 从检索命中详情点击定位时，应跳到对应文档详情并高亮目标切片。

## 3. 实体别名

准备：Admin 在实体别名页添加 `星辰 -> 星辰科技`。

- 查询 `星辰的差旅标准` 应匹配到 `星辰科技`。
- retrieval-test 页面应显示别名匹配 trace。
- 知识查询的检索信息也应显示别名匹配。
- 删除该别名后再次查询，应不再走该 alias；如果仍命中，检查 entity cache invalidate。
- 添加歧义别名，例如 `科技 -> 星辰科技` 和 `科技 -> 远景能源`，查询 `科技的制度` 应显示歧义别名且不强制路由到单一实体。

## 4. 检索策略

在检索测试页分别执行：

- 标准问答：`差旅报销需要哪些审批材料？`
  - 预期：HyDE 开启，RRF 融合，相关性重排，补充上下文。
- 精确查找：`星辰科技的住宿标准是多少？`
  - 预期：HyDE 关闭或预算为 0，结果更少、更精确。
- 全面查找：`电脑丢了应该怎么处理？`
  - 预期：显示 2-4 条扩展查询，每条有命中数；HyDE 不出现。
- 关联查找：`API v1什么时候下线？迁移指南由谁负责？这个人还负责什么工作？`
  - 预期：显示多跳发现 trace，有 hop 信息。

## 5. 知识查询

在知识查询页执行：

- `差旅报销需要哪些审批材料？`
  - 预期：答案有引用，检索信息可展开，引用仍指向 anchor 切片。
- 开启“仅基于资料回答”后，问一个资料中不存在的问题。
  - 预期：回答应更保守，不编造无依据内容。
- 全面查找模式下查询 `报销需要哪些审批材料`。
  - 预期：检索信息显示扩展查询和命中数，引用正常显示。

## 6. 质量中心

- Admin 质量中心概览应显示全局指标、按策略聚合、证据模式。
- Admin 检索记录可按 flavor 筛选。
- 普通用户查询记录页应只显示自己的记录视角。
- 记录详情中的检索命中详情应能通过定位打开对应切片。
- 记录详情应显示阶段耗时、resolved settings、result shape、fallback、token/model 信息。
- 答案反馈可以加入基准测试集草稿；已加入的反馈不应重复加入。
- 回归评测页应显示当前基准测试集、草稿、启用状态和每个 case 的运行结果。
- 回归评测页应能选择：仅检索、轻答案、完整。
- 冒烟集应作为 case 子集筛选，不应和评测模式混淆。
- 运行基准测试集时，进度、通过/警告/失败状态应持续刷新；单个 case 失败不应中断剩余 case。
- 设为基线后，下一次同 mode/flavor 运行应显示 baseline delta。
- 运行任一评测后，系统设置 -> 后台任务应出现对应 eval job。

## 7. 系统设置

- 运行状态能读取聊天模型、Embedding、向量库、数据库、Token 状态。
- 后台任务面板应显示最近文档处理和评测任务；任务状态、进度、message、错误信息可读。
- 策略微调页不同策略下只显示该策略有效的可调项。
- 修改参数后保存，再刷新页面，值应保持。
- 默认情况下不显示标签治理页；设置 `VITE_ENABLE_TAG_GOVERNANCE=true` 后才显示。
- 启用标签治理页时，应显示注册标签、启用状态、UI 可见状态、切片命中数和文档命中数。
- 启用标签治理页时，标签预览支持粘贴文本或选择已有文档，不写入存储。
- 启用标签治理页时，禁用会影响索引内容的标签应提示需要重新处理文档/重建索引。

## 8. 健康检查与基础命令

```bash
curl http://127.0.0.1:8010/health
```

预期：

- `status` 为 `ok`，或在 Milvus 非必需且不可达时显示清晰 warning。
- SQLite `journal_mode` 为 `wal`。
- 数据库、上传目录、解析目录为 writable。
- disk space 和 Milvus 状态可读。

```bash
cd backend
python -m pytest tests/unit -q

python scripts/eval_golden_set.py \
  --golden-set ../data/challenge_golden_set_v1.jsonl \
  --api-base http://127.0.0.1:8010/api \
  --mode retrieval_only \
  --limit 1 \
  --concurrency 1

cd ../frontend
npm run build
```
